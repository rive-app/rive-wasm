Fit.Layout — display:none-on-mount repro

Reproduces two bugs in @rive-app/canvas 2.37.x affecting Fit.Layout consumers whose canvas mounts inside a hidden ancestor:

  1. Bug 2: resizeDrawingSurfaceToCanvas() writes 0 into artboard.width/height when getBoundingClientRect() reports 0×0 (canvas hidden via display:none ancestor).
  2. Bug 1: ResizeObserver fires on every size change, but Rive.onCanvasResize only re-fits on zero ↔ non-zero toggles — non-zero → non-zero deltas (reflow, font load, scrollbar) are silently dropped.

Uses the published @rive-app/canvas@2.37.6 via unpkg, so no local build required. Just npx serve from js/examples/_frameworks/ and open this file.
